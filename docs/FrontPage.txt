


  UAKGQuery представляет собой CORBA сервис доступа к базам данных ,
 обеспечивающий высокоуровневую и эффективную модель взаимодействия
 приложения с источником данных.

  Исторически, UAKGQuery вырос из реализации CORBA Query Service, разработанной
 автором в 1998 году. После анализа типичных сценариев использования сервиса
  с точки
  зрения удобства и производительности, архитектура CosQuery была значительно
  пересмотренна; в результате продукт  выпущен под новым именем и с новой
  функциональностью. Результаты исследований частично обощены в следуйщей 
  <A href="http://www.gradsoft.com.ua/rus/whitepapers/UKRPROG2000/article.html">
статье 
  </A>

 Назначение комплекса:
   -  предоставить универсальный CORBA API для доступа к реляционным базам данных.
   - Обеспечить интеграцию транзакций баз данных в систему распределенных CORBA транзакций. 

 Описание применения 

  Разработчики CORBA приложений используют CORBA API для доступа к БД, которые
транслируются сервисом UAKGQuery в запросы к соответствующей базе данных.


 Требования к программному окружению

  Сервер БД - Oracle на любой платформе. В следующую версию будет внесена
 поддержка InterBase.

  Операционная система со стороны сервера: Sun Solaris, Linux, Windows NT. (возможен быстрый перенос на любую posix-совместимую систему)

  Операционная система со стороны клиента: Sun Solaris, Linux, Windows NT, FreeBSD (возможен быстрый перенос на любую posix-совместимую систему)

  ORB: OmniORB-3.0, ORBacus-4.03, TAO-5.1

  Поддержка транзакций возможна при использовании ORBacus и ORbacus OTS.
