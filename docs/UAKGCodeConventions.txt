
Некоторые Рекомендации по написанию кода


1. Макросы
  - Начинаются двойным подчеркиванием
  - Сложносочиненные названия разделяются одинарным знаком подчеркивания
Пример: __CosQueryFacade_FieldValueAccess_h

2. Переменные (Константы, Экземпляры класса)
  - Начинаюсь со строчной буквы
  - В сложносочиненных переменных каждая часть должна начинаться с заглавной
    буквы
  - Имя должно отражать тип
Пример: nRecords

3. Типы (Имена классов)
  - Начинаются с заглавной буквы
  - В сложносочиненных именах каждая часть должна начинаться с заглавной
    буквы
Пример: UAKGQueryManager

4. Атрибуты класса
  - аналогично переменным
  - должны заканчиваться знаком подчеркивания
Пример: queryManager_

5. Функции (Методы класса)
  - пишутся строчными буквами (исключение - аббревиатуры, устоявшиеся слова)
  - сложносочиненные части разделяются знаком подчеркивания
  - название должно, по возможности, отражать работу с памятью и назначение 
    функции.
Пример: generate_new_str_without_strconsts
