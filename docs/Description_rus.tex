\section{  Введение: }

  UAKGQuery представляет собой CORBA сервис доступа к реляционным базам данных.

\section{ Назначение  комплекса:}

 \begin{enumerate}
   \item Предоставить универсальный CORBA API для доступа к реляционным базам данных.
   \item Обеспечить интеграцию с транзакций баз данных в систему распределенных CORBA транзакций. 
 \end{enumerate}

\section{ Описание применения }

  Разработчики CORBA приложений используют UAKGQueyr CORBA API для доступа к 
БД, путем связывания приложений с клиентскими библиотеками UAKGQuery. Также поддерживается встраивание UAKGQuery в приложения, путем связывания с серверными библиотеками.

\section{ Условия применения }

\subsection{Требования к программному окружению}

 \begin{itemize}
  \item Сервер БД - Oracle-8i,9i , Interbase-6.0 на любой платформе.
  \item Операционная система со стороны сервера: Sun Solaris, Linux, Windows NT. (возможен быстрый перенос на любую posix-совместимую систему)
  \item Операционная система со стороны клиента: Sun Solaris, Linux, Windows NT, FreeBSD (возможен быстрый перенос на любую posix-совместимую систему)
  \item ORB:  ORBacus-4.1, TAO-1.1, TAO-1.2
  \item Поддержка транзакций возможна при использовании ORBacus и ORbacus OTS.
 \end{itemize}

\subsection{Требования к техническому окружению}

 \begin{itemize}
   \item Серверная часть: совпадающие с минимальными требованиями целевой БД
   \item Клиентская часть: совпадающие с минимальными требованиями целевого ORB
 \end{itemize}
